O que são Clients?

(Voltar ao Início)

Clients são dispositivos ou programas que pedem informações ou serviços a outros computadores chamados servidores.
Por exemplo, quando você abre seu navegador e acessa um site, o navegador é o client. Ele envia um pedido ao servidor para receber a página da internet.

Resumo Simples:

  • Cliente é quem pede algo.
  • Pode ser um navegador, um app no celular, ou qualquer dispositivo que acessa serviços pela internet.

O que são Servers?

(Voltar ao Início)

Servers são computadores ou programas que respondem aos pedidos feitos pelos clients, fornecendo os dados ou serviços solicitados.
Por exemplo, o computador onde o site está hospedado é o servidor. Ele recebe o pedido do client e envia a página para o navegador mostrar.
Resumo Simples:

  • Server é quem responde aos pedidos.
  • Ele armazena dados, processa informações e entrega o que o client precisa.

Versionamento de código com Git e Github

(Voltar ao Início)

O que é Git?

Git é uma ferramenta que ajuda programadores a controlar as mudanças feitas no código dos projetos.
Com ele, você consegue salvar versões diferentes do seu trabalho, voltar para versões antigas, e trabalhar junto com outras pessoas sem perder nada.

O que é Git Hub?
GitHub é uma plataforma online onde você guarda seus projetos com Git. Lá, você pode compartilhar seu código, colaborar com outros programadores e acompanhar o progresso do projeto.

Por que usar Git e GitHub?
Porque eles tornam o trabalho em equipe mais organizado, ajudam a evitar erros, e facilitam o desenvolvimento de softwares, mesmo quando várias pessoas estão mexendo no mesmo código.

  • Git init = Inicia um repositório Git em uma pasta. É como começar a usar o controle de versões no seu projeto.
  • Git add = Prepara os arquivos que você quer salvar (ou “commitar”) no controle de versões. É como dizer: “esses arquivos estão prontos para serem salvos”.
  • Git commit = Salva as mudanças preparadas com o git add. Aqui você escreve uma mensagem que explica o que mudou, tipo um registro do que foi feito.
  • Git push = Envia as mudanças salvas no seu computador para um repositório remoto, como o GitHub, para que outras pessoas possam ver e colaborar.
  • Git pull = Puxa as atualizações feitas por outras pessoas no repositório remoto para o seu computador, mantendo tudo sincronizado.
  • Git status = Mostra quais arquivos foram modificados, quais estão prontos para salvar e quais ainda não estão sendo controlados.

Estrutura Básica HTML5

(Voltar ao Início)

HTML (HyperText Markup Language) é a linguagem usada para criar páginas na internet. Toda página HTML tem uma estrutura padrão, que funciona como o esqueleto do site.

  • !DOCTYPE html — informa ao navegador que o documento é HTML5.
  • html lang="pt-BR" — elemento raiz que envolve toda a página, com o idioma definido para português do Brasil.
  • head — área de informações que não aparecem diretamente na página, como o título, codificação de caracteres e links para estilos.
  • title — título que aparece na aba do navegador.
  • body — corpo da página, onde ficam os conteúdos visíveis como textos, imagens, botões, etc.
  • h1 — título principal da página.
  • p — parágrafo de texto.
